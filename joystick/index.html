Come si usa broker, topic, snippet

Topic

Due topic: uno per lo sterzo (leva orizzontale) e uno per la velocità (leva verticale). Puoi anche puntarli sullo stesso e distinguerli con il campo type. Con Sottoscrivi attivo la pagina si rimette in ascolto sui propri topic: gli echi finiscono nel log e servono a misurare l'RTT.

  • Sterzo — angolo della leva orizzontale, intero: {"angle":55} (scala 0..180, dritto 90). È il topic movimento (o steer.topic in config.js).
  • Velocità — posizione della leva verticale e senso di marcia, interi: {"speed":120,"dir":1}. Si imposta in config.js dentro lever.topic.

La riga sopra il log pubblica un payload qualsiasi su un topic qualsiasi: per il resto è un normale client MQTT.

Payload su misura

Tutti i numeri pubblicati sono interi. Con formato Raw il payload è il testo che scrivi, spedito tale e quale. Con formato Personalizzato scrivi tu il payload e i segnaposto vengono sostituiti a ogni invio:

  • {angle} angolo dello sterzo (intero, es. 0..180 con dritto 90) · {heading} left/straight/right
  • {speed} velocità (intero nella scala della leva) · {dir} marcia: 1 avanti, 0 retromarcia
  • {src} sorgente del comando · {t} timestamp ms
sterzo   {"angle":{angle}}          → {"angle":55}
velocità {"v":{speed},"r":{dir}}    → {"v":120,"r":1}
raw      {angle}                    → "55"

Comandi

  • Sterzo (leva orizzontale, a sinistra): trascina, rotellina del mouse, i tasti indicati sotto la leva o lo stick sinistro del gamepad. Con spring torna dritto quando la lasci.
  • Velocità (leva verticale, a destra): trascina, rotellina, i tasti sotto la leva, il grilletto o lo stick destro del gamepad. Resta dove la lasci.
  • Bottoncino avanti/retro sotto la leva velocità (o il suo tasto): commuta dir fra 1 e 0 e ripubblica subito.
  • Spazio = STOP: rimette dritto lo sterzo, azzera la velocità e pubblica lo zero. Anche cambiando scheda o perdendo il focus viene inviato lo zero.

Personalizzare le levette

Tutto quello che riguarda i comandi sta in config.js, accanto alla pagina: nessun build, si salva e si ricarica. steer descrive la leva dell'angolo, lever quella della velocità, defaults i valori con cui parte la pagina (broker, topic, formato, frequenza…).

steer: {
  label: "angolo", unit: "°",
  min: 0, max: 180, center: 90, start: 90, step: 5,
  spring: true,           // torna dritto quando la lasci
  rate: 180,              // gradi al secondo tenendo un tasto
  keys: { left: ["KeyA"], right: ["KeyD"], center: ["KeyS"] },
  gamepadAxis: 0          // stick sinistro, orizzontale
}

La posizione delle leve non viene ricordata fra una sessione e l'altra: alla riapertura si riparte da start, così non ti ritrovi un mezzo che scatta al primo comando.

Le impostazioni cambiate dall'interfaccia restano memorizzate nel browser e vincono su defaults: per tornare al file usa Reset impostazioni.

Consumarli in Python (paho-mqtt)

import json, time, paho.mqtt.client as mqtt

STEER, SPEED = "esp32_car_fralor/move", "esp32_car_fralor/speed"
angle, speed, dir = 0, 0, 1
last = time.time()

def on_message(c, u, msg):
    global angle, speed, dir, last
    d = json.loads(msg.payload); last = time.time()
    if msg.topic == STEER:
        angle = d["angle"]             # intero, es. -90..90
    else:
        speed, dir = d["speed"], d["dir"]
    drive(angle, speed, dir)

c = mqtt.Client()
c.on_connect = lambda c,u,f,rc: c.subscribe([(STEER,0),(SPEED,0)])
c.on_message = on_message
c.connect("broker.hivemq.com", 1883, 30)
c.loop_start()

while True:                            # watchdog: nessun comando da 0.6 s → fermo
    if time.time() - last > 0.6:
        drive(0, 0, 1)
    time.sleep(0.1)

Che porta mettere

Dal browser MQTT viaggia solo su WebSocket: conta la porta WebSocket del broker, non quella TCP. Nel campo host puoi anche incollare un URL intero (wss://host:8884/mqtt): viene spezzato da solo in host, porta, percorso e TLS.

  • broker.emqx.io — 8084 /mqtt TLS (in chiaro: 8083)
  • broker.hivemq.com — 8884 /mqtt TLS (in chiaro: 8000)
  • test.mosquitto.org — 8081 /mqtt TLS (in chiaro: 8080)
  • HiveMQ Cloud / EMQX Cloud — 8884 /mqtt TLS, con username e password del tuo cluster
  • 1883 e 8883 sono MQTT su TCP: dal browser non funzionano

Broker tuo in LAN senza TLS: da questa pagina in HTTPS il browser blocca ws://. Servono un certificato sul broker (o un reverse proxy TLS) oppure aprire questa pagina in locale su http://.

Sicurezza

I broker pubblici sono aperti a chiunque: chi indovina il topic guida il tuo hardware. Per qualcosa di reale usa un broker tuo con TLS, utenza dedicata, ACL sul topic e un watchdog lato dispositivo (se non arriva un move entro N ms, fermati).
