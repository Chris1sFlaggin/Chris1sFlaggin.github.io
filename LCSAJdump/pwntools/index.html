LCSAJGadgets

Zero-import integration that injects LCSAJGadgets into pwntools' top-level namespace. Graph-based gadget discovery available the moment you write from pwn import *.

Available x86_64 · ARM64 · RISC-V64 23 public methods

Exploits

Real-world chains

Three CTF exploits built entirely with the semantic API. Each exposes a structural gap invisible to ropper and ROPgadget.

x86-64
JOP Trampoline Chain
CTFSG 2021 · jop · stack pivot + JOP trampoline
Stack pivot redirects rsp into the JOP chain. A trampoline (add rsp,8; jmp [rsp-8]) acts as a ret substitute — no hardcoded offsets needed.
gadgets = LCSAJGadgets('./jop')
pivot = gadgets.pivot_gadgets()[0]
trampoline = gadgets.trampolines(mnem='add rsp')[0]
zero_rax = gadgets.zero_register('rax')[0]
ARM64
arm_strong
ctfarm · mprotect + shellcode · 3-gadget chain
Format-string leak → canary + stack addr. Three gadgets chain mprotect(stack, RWX) then jump to shellcode. LCSAJGadgets finds the indirect blr x22 and memory-read ldr x3,[x20] dispatchers.
gadgets = LCSAJGadgets('./chal', arch='arm64')
g1 = gadgets.pop_chain('x29','x30')[0]
g2 = gadgets.call_reg('x22')[0]
g3 = gadgets.memory_read('x20','x3')[0]
RISC-V 64
typicalROP
ctfriscv · classic ROP · execve via ecall
Classic multi-gadget ROP chain on RISC-V. A fat gadget loads all argument registers from the stack in one shot, then jumps to ecall for execve("/bin/sh").
gadgets = LCSAJGadgets('./chal', arch='riscv64')
ecall = gadgets.find('ecall')[0]
load_all = gadgets.pop_chain('a0','a1','a7')[0]

Installation

Setup

LCSAJGadgets is not part of pwntools. It is a standalone module that must be manually placed inside the pwntools package directory and registered in pwn/toplevel.py to become available via from pwn import *.

Installazione Automatica

❯ ./install_integrations.sh --pwntools

Questo comando installerà LCSAJdump in modalità "editable" (pip install -e) rendendo la classe LCSAJGadgets importabile globalmente nel tuo sistema.

Uso in Python / Pwntools

from pwn import *
from lcsajdump.integrations.pwntools_helper import LCSAJGadgets

# The ML ranking model is loaded automatically by default
gadgets = LCSAJGadgets('./chal')

# Force classic heuristic ranking (bypass ML)
gadgets_classic = LCSAJGadgets('./chal', algo=True)

When a pre-generated JSON is available, use LCSAJGadgets.from_json('gadgets.json') to skip the re-analysis.

Constructor

LCSAJGadgets()

LCSAJGadgets(
  binary_path: str,
  arch: str = 'auto', # auto · x86_64 · arm64 · riscv64
  bad_chars: str = '', # hex bytes to exclude, e.g. '000a0d'
  depth: int = 20, # max LCSAJ blocks per path
  darkness: int = 5, # per-node visit ceiling
  instructions: int = 15, # max instructions per node
  all_exec: bool = False, # include all executable sections (.coldpath etc.)
)
# from a pre-generated JSON (faster — no re-analysis)
gadgets = LCSAJGadgets.from_json('gadgets_found.json')

API Reference

Methods & Properties

Search

gadgets.find(pattern)

Returns: list[int]

Case-insensitive substring match against the full instruction text. Returns all matching addresses (including duplicates), sorted.

pop_rdi = gadgets.find('pop rdi')[0]
fat = gadgets.find('mov al, 0x3b')[0]

gadgets.best(pattern)

Returns: Gadget | None

Returns the highest-scoring Gadget matching pattern, or None if not found. Use .address to get the integer address.

tramp = gadgets.best('add rsp')

gadgets.address_in_range(low, high)

Returns: list[Gadget]

Filter gadgets to those with primary address in [low, high). Useful when certain ranges contain bad characters, or when narrowing to a specific library after a partial leak.

# only gadgets below 0x410000 (no null bytes in address)
clean = gadgets.address_in_range(0x400000, 0x410000)

gadgets.sets_register(reg)

Returns: list[Gadget]

Gadgets that write to reg (covers mov, ldr, pop, ldp, addi, li patterns). Sorted by score.

g = gadgets.sets_register('rdi')[0].address

gadgets.loads_from_stack(reg)

Returns: list[Gadget]

Gadgets that load reg from the stack (pop, ldr [sp+N], ldp with sp), sorted by score.

GADGET_LOAD_A0 = gadgets.loads_from_stack('a0')[0].address

gadgets.no_clobber(*regs)

Returns: list[Gadget]

Gadgets that do not write to any of the given registers — useful to preserve register state across chain steps.

Trampolines

gadgets.trampolines(mnem=None, pattern=None)

Returns: list[Gadget]

All trampoline gadgets (indirect call/jmp via register). Optionally filter by terminating mnemonic ('call', 'jmp', 'blr'…) and/or instruction substring.

# all trampolines that dispatch via 'call'
gadgets.trampolines(mnem='call')

Exploit Primitives

gadgets.pop_chain(*regs) → list[Gadget]

Gadgets that pop a specific register sequence from the stack. pop_chain('rdi') finds pop rdi; ret.

gadgets.pivot_gadgets() → list[Gadget]

Stack pivot gadgets — redirects rsp/sp to attacker-controlled data. See the Stack Pivot section for category breakdown.

gadgets.zero_register(reg) → list[Gadget]

Gadgets that set reg to zero (xor, li 0, c.li 0…).

gadgets.reg_move(src, dst) → list[Gadget]

Gadgets that copy src into dst (mov, mv, xchg…).

gadgets.write_what_where(what_reg, where_reg) → list[Gadget]

Gadgets that write what_reg into memory pointed to by where_reg.

gadgets.memory_read(src_reg, dest_reg=None) → list[Gadget]

Gadgets that read from memory at src_reg into dest_reg (or any register).

gadgets.call_reg(reg=None) → list[Gadget]

Gadgets that perform an indirect call/blr/jalr via a register (optionally specific one).

gadgets.syscall() → list[Gadget]

Gadgets that perform a system call (syscall, svc, int 0x80, ecall). Useful for finding syscall gadgets across architectures.

gadgets.set_immediate(dst, value=None) → list[Gadget]

Gadgets that load a constant into dst: mov rax, 59 / movz x8, #221 / li a7, 221. Optionally filter by exact value.

gadgets.stack_delta(n) → list[Gadget]

Gadgets whose net rsp/sp adjustment (before the tail) equals n bytes. Positive = stack shrinks. Use stack_delta(8) for x86-64 alignment.

Arithmetic & Logic

gadgets.add_register(dst, src=None) → list[Gadget]

Gadgets containing add dst, src. Optionally filter by source operand. Covers x86, ARM64 (adds), RISC-V.

gadgets.sub_register(dst, src=None) → list[Gadget]

Gadgets containing sub dst, src. Optionally filter by source operand.

gadgets.or_register(dst, src=None) → list[Gadget]

Gadgets containing or/orr/ori dst, src. Covers x86, ARM64, RISC-V.

gadgets.and_register(dst, src=None) → list[Gadget]

Gadgets containing and/ands/andi dst, src. Covers x86, ARM64, RISC-V.

gadgets.xor_register(dst, src=None) → list[Gadget]

Gadgets containing xor/eor dst, src. For zeroing (xor rax,rax) prefer zero_register().

gadgets.shift_register(dst, direction=None, amount=None) → list[Gadget]

Gadgets that shift dst. direction: 'left' / 'right' / 'arith' / 'rot'. Optionally filter by shift count. Covers shl/lsl/slli, shr/lsr/srli, sar/asr/srai, rol/ror.

Chain Planning

gadgets.find_chain(start, end, max_depth=5)

Returns: list

BFS over the gadget graph. Returns up to 3 paths from start to end, shortest first.

Stack Pivot

pivot_gadgets()

A pivot redirects rsp/sp to attacker-controlled memory. The new value must come from a register or memory, not an immediate constant — add rsp, 8, sub rsp, 0x20 and addi sp, sp, -16 are not pivots and are excluded.

Frame-based · x86-64

leave expands to mov rsp, rbp; pop rbp. If you control rbp the chain redirects into your fake frame — the classic ret2leave technique.

# gadget
leave ; ret
pivot = gadgets.pivot_gadgets()[0]

Register-load · x86-64 · ARM64 · RISC-V

rsp is loaded directly from a register the attacker controls. Common in JOP chains where a register was preloaded with the fake stack address.

# x86-64
xchg rsp, rax  ·  mov rsp, rbp
# ARM64
mov sp, x0
# RISC-V
mv sp, a0
pivot = gadgets.pivot_gadgets()[0]
# or: gadgets.best('xchg rsp')

Stack-load · x86-64

pop rsp reads the new rsp off the current stack. You write the target address at the right offset in the overflow buffer — rsp then points into your JOP/ROP chain.

# gadget
pop rsp ; pop rdi ; pop rcx ; ...
# trigger: write fake_rsp at offset 256
payload = flat({256: p64(pivot.address) + p64(rsp + 24)})

Memory-load · ARM64 · RISC-V

rsp/sp is loaded from memory at an address in a register. Predominant on ARM64 and RISC-V where direct register-to-sp moves are less common than on x86.

# ARM64
ldr sp, [x0]
# RISC-V
ld sp, 0(a0)
# attacker controls what x0 / a0 points to
pivot = gadgets.pivot_gadgets()[0]
