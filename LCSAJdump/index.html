Core Features

Beyond Linear Scanning

Multi-Arch Support

Modular design now supporting RISC-V 64GC and easily extendable to x86 and ARM (FORK!).

Graph Reconstruction

Models the binary as a directed graph of Basic Blocks using LCSAJ (Linear Code Sequence and Jump) analysis.

Rainbow BFS

Proprietary backward search algorithm starting from control-flow sinks to reconstruct the chain.

Shadow Gadgets

Identifies complex "Trampoline" chains that bypass bad bytes or "Conditional" branches, impossible to detect with a linear sliding-window.

The Limitation of Linear Scanners

Traditional ROP scanners use a linear (sliding-window) approach over the executable's bytes. While effective for standard sequences, it fails to find executable gadgets crossing non-contiguous memory blocks.

LCSAJdump reconstructs the Control-Flow Graph (CFG) and identifies:

  • Contiguous Gadgets (Linear)

    Standard linear sequences ending with a control-flow transfer.

  • Non-Contiguous (Shadow) Gadgets

    Complex chains spanning multiple Basic Blocks (LCSAJs) to bypass 'bad bytes' via unconditional jumps or branches.

Performance Analysis

Optimal Point: Mathematically Proven

I've been testing LCSAJdump on binaries for every size to find optimal default values combination.

Saturation

How deep do i need to dig?

In the beginning LCSAJdump discovers A LOT OF gadgets pretty fast.

BIG-SIZED BINARIES: ~ 15.000 gadget @ depth 30

MID-SIZED BINARIES: ~ 5.000 gadget @ depth 20-25

LIL-SIZED BINARIES: ~ 20 gadgets @ depth 5

✓ Trepassing saturation point = time waste

Darkness

How dark do i need the nodes to be cut?

Jumps be hiding code. Darkness = how many still-not-visited instructions do i need to discover.

Darkness = 0: Pruned brancehs ~ 22%

Darkness = 1: Pruned brancehs ~ 0%

Darkness ≥ 1: Pruned brancehs = 0%

✓ Basta 1 istruzione "al buio", usiamo 5 per sicurezza (500%)

Efficiency

When do i need to stop?

Time is always increasing, gadgets stop at saturation point.

EXAMPLE:

✅ Depth 30: 15K gadgets in 2s

❌ Depth 50: 15K gadgets in 8s

(Same result, 4x slower!)

✓ Stop at saturation point + 20% = Sweet Spot

Golden Standard

Optimal default values for 99% of CTF cases:

Optimal depth

30-35

Covers 99% of binaries without wasting time. Calculated as: max_depth_saturation + 20% (Heuristic Safety Margin)

Safe Darkness

2-5

200%-500% more of necessary (1). Grants 0 pruned branches.

...SO WHY NOT 1?!

Preserves Path Diversity in omplexed converging flows (e.g. if/else).

Ready to Use

Install LCSAJdump via pip and analyze your binaries in seconds.

Via Pip:

pip install lcsajdump

Basic Scan:

python LCSAJdump.py <path_to_binary>

Parameter Tuning (Arch & Depth):

python LCSAJdump.py -a riscv64 -d 15 -k 100 <path>
-a, --arch: Target architecture (default: riscv64)
-d, --depth: Max search depth (LCSAJ blocks)
-k, --darkness: Pruning threshold (Max visits)
-l, --limit: Max gadgets to display
