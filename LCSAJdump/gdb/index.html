Installation

One-Time Setup

Installazione Automatica

❯ ./install_integrations.sh --gdb

Lo script configurerà automaticamente il tuo ~/.gdbinit e aggiungerà il comando lcsaj a GDB, pwndbg e GEF.

Step 2 — use it

pwndbg> lcsaj --help

Step 2 — add to ~/.gdbinit

# add these two lines to ~/.gdbinit

python import sys; sys.path.insert(0, '/path/to/lcsajdump-debug')

source /path/to/lcsajdump_dbg/integrations/gdb_plugin.py

Replace /path/to/lcsajdump-debug with the actual path to the package root. With a pip install, the plugin file is at the path returned by python -c "import lcsajdump_dbg; print(lcsajdump_dbg.__file__)".

Step 3 — use it

pwndbg> lcsaj --help

# or auto-detect and run immediately

pwndbg> lcsaj

Reference

Command Flags

Opzione Argomento Descrizione
-l, --limit N Number of gadgets to display per section (default: 10)
-b, --bad-chars HEX Hex bytes to exclude from addresses (e.g. 000a0d)
-d, --depth N Max LCSAJ blocks per path (default: 20)
-k, --darkness N Per-node visit ceiling / pruning threshold (default: 5)
-i, --instructions N Max instructions per gadget node (default: 15)
--all-exec - Analyse all executable sections (includes .coldpath)
--find PATTERN Filter gadgets whose instruction text contains PATTERN
--arch ARCH Override architecture: auto · x86_64 · arm64 · riscv64
-al, --algo - Use strictly the algorithmic ranking (bypass ML)
If [binary] is omitted, the plugin auto-detects the inferior from /proc/<pid>/exe or GDB's 'info files' output.

Examples

Run on the auto-detected inferior (most common during a debug session)

pwndbg> lcsaj -l 20

Search for a specific gadget pattern

pwndbg> lcsaj --find "pop rdi"

Analyse all executable sections (finds gadgets in .coldpath, .init, etc.)

pwndbg> lcsaj --all-exec -l 30

Explicit binary, filter bad bytes, find JOP trampoline

pwndbg> lcsaj ./chal -b "000a0d" --find "add rsp"

Cross-arch: analyse an ARM64 binary inside GDB (e.g. with qemu-user multiarch)

pwndbg> lcsaj ./chal_arm --arch arm64 --all-exec

Output

Output Format

Results are split into two sections — sequential (ret-terminated) and jump-based (JOP/call-terminated) — each listing up to -l gadgets with address, instruction text, tag, and heuristic score.

Tag

[ROP] · [JOP] · [lcsaj]

lcsaj marks gadgets exclusive to lcsajdump — not findable by ropper or ROPgadget.

Score

score=230

Optuna-calibrated heuristic score. Higher = more useful for exploit chains on the current architecture.
