# Giving back life to an obsolete IoT (reassessing PogoPlug Pro NAS).

## The Background

While cleaning up my tech drawer, I stumbled upon my old PogoPlug Pro. Once marketed as a revolutionary personal [NAS](https://en.wikipedia.org/wiki/Network-attached_storage), it has essentially become a brick since the official cloud servers were shut down. 

However, beneath its plastic shell, it still has a perfectly capable ARM processor and networking capabilities. Why throw away a piece of [IoT](https://en.wikipedia.org/wiki/Internet_of_things) hardware when we can repurpose it? My goal: flash a custom firmware and turn it into a network-wide **AdBlocker** (similar to a Pi-hole) that protects all devices in my home silently.

---

## Accessing data through hardware with USB-TTL

Since the official software is dead and we can't SSH into it normally, we have to talk directly to the motherboard. To achieve this, we need to interface with the [UART](https://en.wikipedia.org/wiki/Universal_asynchronous_receiver-transmitter) serial port on the board using a [USB-TTL adapter](https://learn.sparkfun.com/tutorials/serial-communication/all).

### Linking GND/TX/RX
![PogoPlus](/images/PogoPlus.jpeg)

I opened the case and used a multimeter to identify the ground (`GND`), transmission (`TX`), and reception (`RX`) pins on the debug header.

|USB-TTL|POGO|
|---|---|
|GND|GND|
|TX|RX|
|RX|TX|

Instead of relying on the broken stock OS, we are going to install [OpenWrt](https://openwrt.org/), a highly extensible Linux operating system for embedded devices.

### Interrupting the Bootloader with CuteCom GUI

### Sharing firmware from my PC to my Pogo's RAM

Since the PogoPlug currently has no working OS to download files, we need to push the firmware directly into its RAM over the network using [TFTP (Trivial File Transfer Protocol)](https://en.wikipedia.org/wiki/Trivial_File_Transfer_Protocol).

I attached a MALE-MALE ethernet cable directly from my PC to the PogoPlug and configured their IPs to be on the same subnet.

```manjaro
# My PC setup
sudo ip addr flush dev eno1
sudo ip addr add 192.168.33.2/24 dev eno1
sudo ip link set eno1 up

```

```cutecom
# PogoPlug setup via serial
CE>> setenv ipaddr 192.168.33.195
CE>> setenv gatewayip 192.168.33.2
CE>> setenv serverip 192.168.33.2

```

Here is the link to the firmware I used: [OpenWrt Oxnas (initramfs)](https://downloads.openwrt.org/releases/23.05.6/targets/oxnas/ox820/openwrt-23.05.6-oxnas-ox820-cloudengines_pogoplugpro-initramfs-uImage)

```manjaro
# Copy initramfs uImage file to tftp buffer and start TFTP server
sudo cp openwrt-23.05.6-oxnas-ox820-cloudengines_pogoplugpro-initramfs-uImage /srv/tftp/uImage
sudo in.tftpd -L -s /srv/tftp

```

```cutecom
# Download the firmware into the Pogo's RAM and boot it
CE>> tftpboot 64000000 uImage
CE>> bootm 64000000

```

### Making this change permanent (Flashing to NAND)

Right now, OpenWrt is running entirely in RAM. If we pull the plug, everything disappears. To make it permanent, we must flash the `sysupgrade` image into the internal [NAND Flash Memory](https://en.wikipedia.org/wiki/Flash_memory%23NAND).

First, let's connect both the PogoPlug and my PC to the standard home router so they have internet access.

```cutecom
root@OpenWrt:/# /etc/init.d/network restart
root@OpenWrt:/# ifconfig
br-lan    Link encap:Ethernet  HWaddr 12:2C:C5:4C:E3:29  
          inet addr:192.168.1.29  Bcast:192.168.1.255  Mask:255.255.255.0
          ...

```

Then, we start a simple HTTP server on my PC to serve the final installation file. Link to the sysupgrade I used: [sysupgrade squashfs](https://downloads.openwrt.org/releases/23.05.6/targets/oxnas/ox820/openwrt-23.05.6-oxnas-ox820-cloudengines_pogoplug-series-3-squashfs-sysupgrade.tar)

```manjaro
# Start an HTTP server on PC
python3 -m http.server 8080 --bind 0.0.0.0

```

```cutecom
# Download and flash from the PogoPlug
cd /tmp
root@OpenWrt:/tmp# wget --no-check-certificate "[http://192.168.1.](http://192.168.1.)X:8080/openwrt-23.05.6-oxnas-ox820-cloudengines_pogoplugpro-squashfs-sysupgrade.tar"

root@OpenWrt:/tmp# sysupgrade -n /tmp/openwrt-23.05.6-oxnas-ox820-cloudengines_pogoplugpro-squashfs-sysupgrade.tar

```

Once flashed, we reboot and setup the bootloader to always launch our newly installed OS.

```cutecom
reset

```

```cutecom
setenv bootcmd 'nboot 60500000 0 440000; bootm'
setenv serverip 192.168.1.2
setenv ipaddr 192.168.1.1
saveenv

```

And just like that, we have a fully functional OpenWrt Linux machine!

## Activating our Local AdBlocker

Now for the fun part. Since we have a mini-server running 24/7, we can use [DNSmasq](https://en.wikipedia.org/wiki/Dnsmasq) (a lightweight DNS forwarder) to block ads and tracking domains for every device connected to our home network.

### The Automation Script

Instead of manually editing files on the PogoPlug, I wrote a bash script that pushes the configuration via SSH. It fetches the blocking lists from StevenBlack and uBlock, formats them, restarts the DNS service, and even creates a tiny web dashboard to check the status!

```bash
#!/bin/bash
POGO_IP="192.168.1.12"

ssh -T root@$POGO_IP << 'EOF'
  echo "=> 1. Cleaning and Preparation..."
  /etc/init.d/dnsmasq stop 2>/dev/null
  killall -9 dnsmasq 2>/dev/null
  rm -f /var/run/dnsmasq* /tmp/adblock.hosts
  mkdir -p /www/cgi-bin

  echo "=> 2. Download Lists (StevenBlack + uBlock)..."
  echo "nameserver 8.8.8.8" > /tmp/resolv.conf.auto
  ln -sf /tmp/resolv.conf.auto /etc/resolv.conf
   
  # Low size HOSTS formatting
  wget --no-check-certificate -qO- "[https://raw.githubusercontent.com/StevenBlack/hosts/master/hosts](https://raw.githubusercontent.com/StevenBlack/hosts/master/hosts)" | awk '/^0\.0\.0\.0/ && $2 != "0.0.0.0" {print "0.0.0.0 "$2}' > /tmp/adblock.hosts
   
  wget --no-check-certificate -qO- "[https://raw.githubusercontent.com/uBlockOrigin/uAssets/master/filters/filters.txt](https://raw.githubusercontent.com/uBlockOrigin/uAssets/master/filters/filters.txt)" | grep "^||" | cut -d"^" -f1 | sed 's/||/0.0.0.0 /' >> /tmp/adblock.hosts

  sort -u /tmp/adblock.hosts -o /tmp/adblock.hosts

  echo "=> 3. Creating Dashboard CGI..."
  cat << 'DASH' > /www/cgi-bin/dash.sh
#!/bin/sh
echo "Content-type: text/html"
echo ""
BLOCKED=$(wc -l < /tmp/adblock.hosts 2>/dev/null | awk '{print $1}' || echo "0")
IP_ADDR=$(uci get network.lan.ipaddr 2>/dev/null || echo "192.168.1.12")
echo "<html><head><meta charset='UTF-8'><meta name='viewport' content='width=device-width, initial-scale=1'>"
echo "<title>Pogo Shield</title><style>
  body{background:#0a0a0a;color:#eee;text-align:center;font-family:sans-serif;padding-top:80px;}
  h1{font-weight:300;display:flex;align-items:center;justify-content:center;gap:15px;}
  .num{font-size:85px;color:#3498db;font-weight:bold;margin:10px 0;text-shadow: 0 0 20px rgba(52,152,219,0.3);}
  p{color:#888; letter-spacing:1px;}
  .dns-info{margin-top:40px; color:#eee; font-size:1.1em;}
  hr{border:0;border-top:1px solid #222;width:250px;margin:30px auto;}
</style></head><body>"
echo "<h1>Pogo-Shield Is Active</h1>"
echo "<div class='num'>$BLOCKED</div>"
echo "<p>BLOCKED ADVERTISING DOMAINS</p>"
echo "<hr><div class='dns-info'>Set DNS on: <b style='color:#3498db;'>$IP_ADDR</b></div>"
echo "</body></html>"
DASH

  chmod +x /www/cgi-bin/dash.sh
  echo "<html><head><meta http-equiv='refresh' content='0; url=/cgi-bin/dash.sh' /></head></html>" > /www/index.html

  echo "=> 4. Running DNS and Web Server..."
  dnsmasq -q -p 53 -S 8.8.8.8 -x /var/run/dnsmasq.pid -C /dev/null --addn-hosts=/tmp/adblock.hosts --cache-size=2000 &
  
  /etc/init.d/uhttpd restart >/dev/null 2>&1
  
  sleep 2
  if netstat -tulnp | grep -q :53; then
      echo "=> SUCCESS! dnsmasq is executing."
  else
      echo "=> CRITICAL ERROR! dnsmasq is crashed."
  fi

  echo "BLUE SHIELD IS ACTIVE!"
EOF

```

I wrote this script to download adblockers from just 2 lists (StevenBlack and uBlock) to ensure we don't overload the PogoPlug's limited memory. You can add more, but keep resource constraints in mind!

### Router DNS Configuration

The very last step was opening my main home router's settings (gateway) and changing the **Primary DNS Server** to the PogoPlug's IP Address (`192.168.1.12` in my case). Now, every time my Smart TV, smartphone, or PC tries to load an ad, the PogoPlug intercepts the request and safely blocks it!

# Ridare vita ad un IoT obsoleto (rivalutando PogoPlug Pro NAS).

## Il Background

Mentre riordinavo i cassetti della tecnologia in camera, mi sono imbattuto nel mio vecchio PogoPlug Pro. Un tempo venduto come un rivoluzionario [NAS](https://it.wikipedia.org/wiki/Network_Attached_Storage) personale, è diventato praticamente un fermacarte da quando i server cloud ufficiali sono stati spenti.

Tuttavia, sotto la sua scocca di plastica, ha ancora un processore ARM perfettamente funzionante. Perché buttare un pezzo di hardware [IoT](https://it.wikipedia.org/wiki/Internet_delle_cose) quando possiamo reinventarlo? Il mio obiettivo: flashare un firmware custom e trasformarlo in un **AdBlocker di rete** (simile a un Pi-hole) che protegga tutti i dispositivi di casa in modo del tutto invisibile.

---

## Accediamo ai dati tramite l'hardware con un adattatore USB-TTL

Poiché il software ufficiale è morto e non possiamo più collegarci via SSH normalmente, dobbiamo "parlare" direttamente con la scheda madre. Per farlo, dobbiamo interfacciarci con la porta seriale [UART](https://it.wikipedia.org/wiki/UART) usando un adattatore [USB-TTL](https://it.wikipedia.org/wiki/Trasmettitore-ricevitore_asincrono_universale).

### Colleghiamo GND/TX/RX

Ho aperto il case e usato un multimetro per identificare i pin ground (`GND`), trasmissione (`TX`) e ricezione (`RX`) presenti sulla scheda.

| USB-TTL | POGO |
| --- | --- |
| GND | GND |
| TX | RX |
| RX | TX |

---

Invece di affidarci al vecchio sistema operativo rotto, installeremo [OpenWrt](https://openwrt.org/), un potentissimo sistema operativo Linux pensato appositamente per i dispositivi di rete integrati.

Ora stacchiamo il cavo che avevamo collegato prima e attacchiamo sia il Pogo che il PC al router di casa, in modo che abbiano accesso a internet.

Terminato il processo, riavviamo la macchina e impostiamo le variabili del bootloader in modo che parta sempre il nostro nuovo sistema.



```bash
  /etc/init.d/uhttpd restart >/dev/null 2>&1
  
  sleep 2
  if netstat -tulnp | grep -q :53; then
      echo "=> SUCCESS! dnsmasq is executing."
  else
      echo "=> CRITICAL ERROR! dnsmasq è crashed."
  fi

  echo "BLUE SHIELD IS ACTIVE!"
EOF

```

In questo script scarico liste di adblock da solo 2 provider (StevenBlack e uBlock) per assicurarmi di non sovraccaricare la memoria limitata del PogoPlug. Se preferisci puoi usarne altre o aggiungerne, ma ricorda sempre di tenere d'occhio i vincoli hardware!

### Configurazione del Router DNS

Il mio prossimo e ultimo passo è stato quello di accedere al pannello di controllo (gateway) del mio router di casa e impostare l'indirizzo IP del Pogo (`192.168.1.12` nel mio caso) come **Server DNS Primario**. Da questo momento, ogni volta che la TV, il telefono o il PC provano a caricare una pubblicità, il PogoPlug intercetta la richiesta e la blocca a monte!
